# 查询类接口的输入输出

数据集、图层、地图都有两个查询接口:

# Query

传入查询条件表达式 (opens new window)，返回符合查询条件的数据

# Nearest

传入一个点，返回距离这个点最近的n条数据，也支持用条件表达式 (opens new window) 筛选后在结果中返回最近的n条数据

# 输入参数

绝大部分接口都包含两个输入参数：表达式expression、表达式绑定的参数bindParams。

# expression

expression是按mapbox条件表达式书写的字符串，仅实现了与要素筛选相关的lookup decision string math类型，并增加了spatial类型用于计算空间关系。

lookup decision string math的用法与mapbox官方文档 (opens new window)一致。

spatial类型新增了四个函数：

# bboxIntersects

判断输入的bbox是否与要素的geometry相交(envIntersects)

Syntax

[
  "bboxIntersects",
  [
    xmin,
    ymin,
    xmax,
    ymax
  ]
]: boolean

示例

[
  "bboxIntersects",
  [
    90,
    20,
    92.5,
    21.3
  ]
]

# geoIntersects

判断输入的wkt geometry是否与要素的geometry相交

Syntax

[
  "geoIntersects",
  wkt_string
]: boolean

示例

[
  "geoIntersects",
  "LINESTRING(100 20,120 30)"
]

# bboxIntersection

输入bbox，若geometry与要素相交则裁剪要素的geometry并返回裁剪后的要素，若不相交则返回null

Syntax

[
  "bboxIntersection",
  [
    xmin,
    ymin,
    xmax,
    ymax
  ]
]: boolean

示例

[
  "bboxIntersection",
  [
    90,
    20,
    92.5,
    21.3
  ]
]

# geoIntersection

输入geometry，若geometry与要素相交则裁剪要素的geometry并返回裁剪后的要素，若不相交则返回null

Syntax

[
  "geoIntersection",
  wkt_string
]: Feature

示例

[
  "geoIntersection",
  "LINESTRING(100 20,120 30)"
]

# bindParams

绑定参数是一个json字符串，可用绑定参数对expression进行参数绑定。

例如表达式["concat", "$a","$b"]，对其传入绑定参数{"$a":"hello","$b":"world"} 时，表达式被解析为["concat", "hello","world"]，表达式输出结果"helloworld"。

换言之，所有$开头的字符串都会被认定为绑定参数，如果字符串的真实值确实以$开头，可通过绑定参数传入具体值来解决， 例如，希望表达式["concat", "$a","sss"]输出结果为"$asss"，则可通过传入参数{"$a":"$a"}来解决。

使用参数绑定可减少解析时的性能消耗。

# 返回值

返回值支持两种格式：geojson (opens new window)和ProtoFeature，默认格式是geojson

其中，ProtoFeature是giscat基于protobuf (opens new window) 设计的一种针对feature对象的压缩规范，其message定义文件在这里 (opens new window) 。 在数据量较大时，ProtoFeature比geojson有更高的压缩率和性能。同时giscat配备了ProtoFeature字节码还原为json对象的java工具 (opens new window) 以及js工具 (opens new window)，方便大家快捷地在java和js中使用。
